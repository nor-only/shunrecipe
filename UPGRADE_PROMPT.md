# SHUNRECIPE v2 — リッチ・ハイグレード化 改修指示書

対象リポジトリ: `shunrecipe`（Vercel 公開中: https://shunrecipe.vercel.app/ ）
依頼者: 奥田（フューチャーテック株式会社 / AIさんいんプロジェクト）
作業モード: Claude Code（VS Code 連携 / GitHub 直結）

---

## 0. 改修の目的（Why）

現公開サイトは情報量が少なく、装飾も絵文字アイコン＋黒帯＋単調なカードの多用で「AI が量産した感」が強い。以下 4 点を同時に解消し、**編集誌（エディトリアル）のような品格と、データベースとしての実用性を両立した旬野菜 × レシピのリファレンス・メディア**へ引き上げる。

1. **情報の厚み**：全国 8 地域 × 四季 × 主要野菜を網羅する構造化データに刷新する。
2. **データベース挙動**：季節 × 地域 × キーワードで高速に絞り込める、サイト内検索 UI を実装する。
3. **ノン AI デザイン**：絵文字アイコン・黒帯サブタイトル・量産型カードを全廃し、余白・タイポグラフィ・罫線で魅せるエディトリアル UI へ。
4. **ヒーローの世界観**：静止画のヒーロー背景を、**ゆったりとしたクロスフェードのランダム・スライドショー**に変更する。

> 全体トーン: *Kinfolk* × *dancyu* × *minimalissimo* — 静謐・余白・活版。

### 2-2. 検索 UI（データベース挙動）

#### 配置と仕様

- **場所**: ヒーロー直下の固定位置（スクロール追随はしない）。`/seasonal/` `/recipes/` でも共通のフィルタバーを使い回す。
- **フィルタ項目**:
  - Season: `すべて / 春 / 夏 / 秋 / 冬`（Pill Button、シングル選択）
  - Region: `全国 / 北海道 / 東北 / 関東 / 中部 / 近畿 / 中国 / 四国 / 九州・沖縄`（Pill Button、シングル選択）
  - Genre（レシピのみ）: `和 / 洋 / 中 / エスニック / 創作`（Pill、マルチ選択可）
  - Time（レシピのみ）: `〜15min / 〜30min / 〜60min / 60min〜`（セグメント）
  - Search 入力: 野菜名・レシピ名・産地・タグに対して全文インデックス検索（部分一致、かな/カナ/ローマ字揺れ許容）。
- **挙動**:
  - 入力即時反映（デバウンス 150ms）、件数カウンタ表示（例: `24 件 / 全 78 件`）。
  - URL クエリ同期（`?season=winter&region=kinki&q=九条ねぎ`）→ シェア・ブックマーク可能。
  - ヒットゼロ時は「該当なし」ではなく **近隣地域・近接シーズンの候補を 3 件サジェスト** する編集者気質の挙動に。
- **技術実装**:
  - Astro Content Collections で vegetables / recipes を読み込み、ビルド時に `public/search-index.json` を生成。
  - クライアントは軽量な全文検索（**Fuse.js** または **MiniSearch**）で処理。サーバレス API 不要。
  - 検索入力は `role="search"` + `aria-live="polite"` でアクセシブルに。

### 2-3. デザイン刷新（ノン AI 化）

#### 排除するもの（必ず消す）

- [ ] セクションタイトル上の「黒い帯のサブタイトル」（`In Season Now` `Editor's Selection` 等の帯背景）
- [ ] 絵文字アイコン（🎋 🥬 🌿 🌼 🧅 🌸 ☀️ 🍁 ❄️ など **すべて**）
- [ ] Material Icons の多用（`arrow_forward` `schedule` `local_fire_department` `restaurant` `favorite` `location_on` `expand_more` 等）
- [ ] 全カードに共通して敷かれる薄グレーのラウンド矩形背景
- [ ] 中央寄せ＋ドロップシャドウの多用

#### 置き換えの方針

| 要素 | Before（AI感） | After（エディトリアル） |
|---|---|---|
| セクションタイトル | 黒帯の上に英小見出し「In Season Now」 | **セリフ体の日本語大見出し**＋下に短い ruled line（0.5px）と章番号（*Chapter 01 — 春の旬*） |
| カード | 角丸 16px のグレー背景 | 背景なし。**画像 → 細い罫線 → タイポグラフィ**の縦組み。画像は 4:5 か 3:4 の縦長で統一。 |
| アイコン | 絵文字・Material Icons | **Phosphor Icons の Thin / Duotone**、もしくは **lucide Icons の stroke 1px** のみ使用。野菜カードでは**アイコンを使わず、野菜名の活字タイポグラフィのみ**で見せる。 |
| ボタン | 黒背景ピル | **下線アニメーションの in-line リンク**（`Read more ——→`）、もしくは罫線 1px の枠線ボタン（ホバーで反転）。 |
| セクション区切り | 余白のみ | **段落罫線（hairline 0.5px）＋章番号**で誌面感を演出。 |

#### タイポグラフィ

```css
--font-serif-jp: "Shippori Mincho B1", "Noto Serif JP", serif;   /* 見出し・本文 */
--font-sans-jp: "Noto Sans JP", sans-serif;                      /* UI のみ */
--font-serif-en: "Fraunces", "Cormorant Garamond", serif;        /* 英字アクセント */
--font-mono: "JetBrains Mono", ui-monospace;                     /* 章番号・メタ情報 */
```

- 本文は **Shippori Mincho B1 17〜18px / line-height 1.9 / letter-spacing 0.04em**。
- 英字キャプションは Fraunces italic の small caps。
- **全ページの文字サイズを現行より 1〜2 段階大きく**し、読み物として成立させる。

#### カラーパレット（ミニマル・土の色）

```css
--bg: #FAF8F3;           /* 生成り */
--ink: #1C1A17;          /* 墨色（真っ黒を避ける） */
--ink-2: #4A463F;        /* 副文 */
--line: #D9D3C3;         /* 罫線 */
--accent: #6B4423;       /* 焙じ茶色 */
--accent-2: #A8432E;     /* 朱 */
--sakura: #E8C3B9;       /* 春 */
--leaf: #6B8E4E;         /* 夏 */
--koyo: #B45B2B;         /* 秋 */
--yuki: #7A8A99;         /* 冬 */
```

季節ごとにアクセント色を切り替える（ヘッダ罫線・章番号のみ色付け、面では使わない）。

#### レイアウト

- 最大幅 **1240px**、本文段組は **2 カラム（12 + 8 グリッド）** を積極的に使う。
- カードグリッドは 3 カラム固定をやめ、**2〜4 カラムの非対称グリッド**（最初の 1 枚だけ大きい誌面レイアウト）を導入。
- モバイルでも文字と罫線の品を落とさない（見出しは縦書きも検討）。

### 2-4. ヒーロー改修 ─ ランダム・クロスフェード

#### 仕様

- 背景画像 **5〜8 枚**をランダム順に無限ループで表示。
- **1 枚の表示時間: 7〜9 秒**（ランダム揺らぎ可）、**フェード所要: 1.8〜2.2 秒**。
- `opacity` と `transform: scale(1.00 → 1.05)` を同時にかけた **静かな Ken Burns 演出**（移動量は控えめに）。
- `prefers-reduced-motion: reduce` を尊重 → アニメーション停止、最初の 1 枚を固定表示。
- 画像は **2560×1440 の WebP / AVIF**、LQIP（`blurhash` or 20px blur）でプリロード演出。
- テキスト可読性のため、画像の上に **縦方向グラデーション（`#1C1A17` 透過）** を 1 枚乗せる（濃すぎないこと）。

#### 実装指針

```ts
// Astro + Vanilla JS で OK
const imgs = shuffle([...]);     // ビルド時に import.meta.glob で収集
let i = 0;
setInterval(() => {
  i = (i + 1) % imgs.length;
  // 2 枚重ねた <div class="hero-bg"> の opacity をクロスフェード
}, 8000);
```

採用候補画像（新規に用意して `public/images/hero/` へ）:
- 朝霧の棚田
- 土付きの根菜を並べた板の間
- 市場の光、木箱の野菜
- 雨上がりの畑の葉
- 土間と籠と包丁の静物
- 湯気の立つ土鍋
- 藁苞に入った納豆や味噌樽（和の道具）
- 台所の窓辺、器に盛られた旬の一皿

> AI 生成画像は避け、**Unsplash / Pexels / pixta / 写真 AC** から無償 or 購入ライセンスで厳選。被写体のトーンを 3〜4 系統に揃え、ヒーロー内で色温度が暴れないようにする。

### 2-5. その他の品質向上

- **構造化データ**: 各レシピに `schema.org/Recipe` JSON-LD を必ず埋め込む（検索流入狙い）。
- **OGP**: ページ種別ごとに OGP 画像テンプレート（タイトル＋縦罫線＋章番号）を自動生成。
- **サイトマップ / RSS**: Astro 標準機能で生成。
- **パフォーマンス**: Lighthouse で Performance / Accessibility / Best Practices / SEO すべて **95+**。
- **アクセシビリティ**: `prefers-reduced-motion`、キーボード操作、コントラスト AA 以上、`role="search"` 等を徹底。
- **フッター**: 現行の運営元表記（AIさんいんプロジェクト）は維持。版表記は `Edition · 2026 / [YYYY] / [MM] — No. 00X` のナンバリングを維持。

## 4. 受け入れ基準（Definition of Done）

- [ ] 全野菜ページ・レシピページが上記スキーマの構造化データで統一されている
- [ ] 季節 × 地域 × キーワード × ジャンル × 時間で URL クエリ同期フィルタが動作する
- [ ] サイト内から絵文字アイコン・黒帯サブタイトル・Material Icons（`arrow_forward` 系）が**完全に消えている**
- [ ] ヒーロー背景がランダム順 7〜9 秒周期でクロスフェードしている／`prefers-reduced-motion` で停止する
- [ ] レシピが **16 件以上**、野菜が **60 件以上**収録されている
- [ ] Lighthouse 4 項目がすべて 95+
- [ ] PR 本文に Before / After のスクリーンショットが添付されている

---

## 5. 実装順序の推奨

1. **デザイン基盤**: CSS 変数、タイポ、罫線コンポーネント（`<Rule />`, `<ChapterHeading />`, `<EditorialCard />`）を先に作る。
2. **ヒーロー改修**: 新コンポーネント `<HeroSlideshow />` を実装、既存ヒーローを差し替え。
3. **コンテンツ層刷新**: `vegetables/` `recipes/` を YAML スキーマに合わせて再生成。
4. **検索 UI**: ビルド時 index 生成 → `<SearchBar />` + `<FilterPills />` を 3 ページに展開。
5. **黒帯・絵文字・Material Icons の一掃**: 全ページを grep で走査し、残存ゼロを確認。
6. **メタ・OGP・JSON-LD・a11y・パフォーマンス**を仕上げて PR。

---

## 6. 禁止事項

- AI 生成画像をヒーロー背景・レシピ写真に使わないこと。
- 絵文字を「装飾」として再導入しないこと（本文内での引用は可）。
- 黒背景の帯・丸アイコンチップを復活させないこと。
- 「美味しさの秘密」「3 つのポイント」など量産ブログ定型句を使わないこと（必ず誌面的な本文に書き直す）。
